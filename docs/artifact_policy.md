# Artifact Policy

## Track in Git

- source code under `scripts/`
- tests under `tests/`
- durable model manifests and decision artifacts under `models/`
  - examples: `*_summary.json`, `*_report.json`, `core_champion.json`
- docs under `docs/`

## Do Not Track in Git

- runtime state snapshots
  - `models/rotation_target_050_live_state.json`
  - `models/pairwise_regime_live_state.json`
  - `models/pairwise_regime_shadow_state.json`
  - `models/pairwise_regime_mixture_shadow_live_state.json`
  - keep example schemas instead, such as `models/rotation_target_050_live_state.example.json`
- operational health snapshots
  - `models/operation_health_report.json`
  - `models/operation_health_history.jsonl`
- runtime logs
  - `logs/*.log`
  - `logs/*.jsonl`
- local caches and generated bytecode
  - `data/market_context/`
  - `data/binance_futures/*_1d.csv`
  - `data/binance_futures/*_5m.csv`
  - `data/binance_futures/*_funding_*.csv`
  - `data/binance_futures/derivatives/`
  - `scripts/__pycache__/`

## Review Queue

Files generated by experiments but not yet accepted into the durable record should remain untracked until reviewed.

- stop-loss compare outputs
- new walk-forward or GA curve CSVs
- rerun stress reports used for decision support

This keeps the repository clean while preserving important research artifacts once they are approved.
